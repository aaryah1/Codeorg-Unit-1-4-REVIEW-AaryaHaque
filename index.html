Unit 2

The internet isn’t just made out of thin air, you need specific tools to be able to access the internet or you need specific technology. Here are different aspects of the internet you need to know in order to understand how the internet exactly works. The computing network is the group of devices that data is received and sent on. Behind that is called a computing system where there are devices and programs working together for a common purpose. The network are the actual devices that are included in the system. Just like us the internet had to follow rules, standards, and regulations. This is called protocol. The internet protocol is transferring data across the internet that assigns a unique number to each connected device. This unique number is called an IP address. This IP address is what identifies the device just like your house address because no other house has that address. Before we jump into the data that gets sent and received let’s talk about the bandwidth and how it gets there. The bandwidth is the maximum amount of data that can be sent in a fixed amount of time. The data then travels through the network within paths and inside packets. Packets are little broken pieces of data that get sent in packages. Since there can be a very large amount of data it can’t all be sent at once so it needs to be broken down into little packets and sent individually so there isn’t a system override. Adding onto the packets are something called packet metadata which is additional data added on to guide them through the network and assemble the original message when it gets to its receiver. Remember how I talked about protocols before? There are different protocols for each part of the internet. For sending the packets of information we have TCP which stands for Transmission Control Protocol which does all error checking to ensure all packets are received and properly ordered. UDP stands for User Datagram Protocol which is a protocol that sends the packets quickly and doesn’t worry much about error checking. Now that you know how data is sent, let's talk about how we translate domain names like “example.com” into IP addresses to receive specific data. We use something called DNS which stands for Domain Name System to translate the domain anime into the unique numbers for computing devices (Ip address). There’s a whole system of DNS servers that work together and communicate with one another to translate the domain names into IP addresses. DNS servers are in a distributed hierarchy and DNS associates the website name with the IP address so your computer uses DNS to translate the domain name to an IP address. Since DNS is in a distributed hierarchy this scales out the internet since it’s always expanding. We know that computers communicate but how do they communicate? They communicate using HTTP which stands for Hypertext Transfer Protocol. This involves what we call a “GET request” which is when the computers request information from another computer. All data gets transferred through the server to the computers which is called the “physical connection.”
